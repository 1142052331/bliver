# 集合图钉与直达列表设计

日期：2026-07-14

## 背景

当前 `ClusterMarkers` 使用一个圆形数字表示集合。数字只能说明足迹数量，无法表达集合是由多个地点构成、包含哪些来源或是否有未读内容。

点击集合后，现有逻辑会先判断坐标是否完全重合或地图是否接近最大缩放级别。只有满足条件时才打开列表，否则执行 `fitBounds`。密集区域会经历“集合图钉 -> 缩放后的集合图钉 -> 再次点击 -> 单图钉”的多层操作，用户无法预期一次点击会得到列表还是缩放。

## 目标

1. 用叠放图钉替代裸数字圆球，保留明确的地图语义。
2. 集合图钉稳定显示去重后的地点数，例如 `3 个地点`。
3. 第一次点击集合图钉直接打开集合足迹列表，不再强制缩放。
4. 列表中保留可选的“在地图中展开”，让空间分布成为辅助路径。
5. 保留现有来源色、未读状态、单图钉点击和足迹详情行为。

## 非目标

- 不更改服务端地图查询或聚类数据量。
- 不替换 `leaflet.markercluster`。
- 不重新设计单个足迹图钉。
- 不重做足迹详情页、地图筛选或搜索。
- 不迁移或删除当前没有应用入口的旧 `ClusterDetailPanel`。

## 方案选择

采用“叠放图钉 + `N 个地点` 标签 + 首次点击直达列表”。

没有采用头像环，因为它会优先表达人物而弱化地点关系，并且匿名或无头像数据会降低稳定性。没有采用照片拼贴，因为照片不是每条足迹都具备，地图缩放较小时图片也难以辨认。叠放图钉与地图语义一致，使用现有来源色即可表达集合构成。

点击行为采用直接打开列表。预览气泡会增加一次操作；自动缩放再显示列表仍可能产生二次集合，无法保证点击路径。

## 集合描述数据

新增纯函数 `buildClusterDescriptor(markers)`，从 Leaflet 子 marker 元数据生成稳定描述：

```text
placeCount       去重后的有效坐标数量
footprintCount   集合中的足迹总数
sourceScopes     按出现频率排序、最多三个的来源类型
hasUnread        是否至少包含一条未读足迹
label            `${placeCount} 个地点`
accessibleLabel  `${placeCount} 个地点，${footprintCount} 条足迹[，包含未读更新]`
```

每个 marker 除现有 `_footprintId`、`_isUnread`、`_sourceScope` 外，继续使用 `getLatLng()` 提供有效坐标。坐标按经纬度各保留六位小数生成去重键，与现有“近似相同坐标”的容差一致，避免浮点噪声把同一地点拆开。

来源按集合内出现次数从高到低选择，数量相同则使用固定优先级：本人、好友、同省、同国、全球。这样数据刷新前后图钉叠放顺序不会随机跳动。

## 图标视觉

集合图标由最多三枚小图钉叠放组成：

- 第一枚为主要来源色，位于前景；
- 第二、第三枚使用次要来源色并轻微旋转、错位；
- 单一来源时三枚图钉使用同一来源色的明暗变化，不制造不存在的来源；
- 混合来源沿用本人、好友、同省、同国和全球的现有颜色变量；
- 存在未读足迹时，右上角显示一个未读圆点；
- 图钉下方使用白底小标签显示 `N 个地点`，不显示裸数字圆球。

外层 Leaflet 图标保持至少 48×48px 的点击区域。图形和标签使用固定布局约束，标签不换行；较大的地点数仍显示完整数字，图标只叠放三枚，不随数量增高。图标通过 `role`、`aria-label` 或 Leaflet 可访问标题暴露完整 `accessibleLabel`。

`buildClusterHtml(descriptor)` 负责 HTML 生成并对所有动态文本转义。缓存键包含地点数、足迹数、来源序列和未读状态，确保视觉状态变化后不会复用旧图标。

## 点击与状态流

`ClusterMarkers` 的 `clusterclick` 不再调用 `shouldOpenSamePlace` 或自动 `fitBounds`，而是立即生成选择对象并写入 UI Store：

```text
clusterData = {
  footprintIds,
  bounds,
  placeCount,
  footprintCount
}
```

`clusterData` 已存在但未被应用使用，本次将它明确为当前集合选择。原 `samePlaceIds`、`openSamePlace` 和 `closeSamePlace` 被迁移为 `openCluster`、`closeCluster`，避免“同一地点”名称继续承载多个地点集合。

`MapView` 在 `MapContainer` 内渲染新的 `ClusterFootprintSheet`。面板可通过 Portal 显示在页面顶层，但仍保留 Leaflet map context，因此“在地图中展开”可以直接调用 `map.fitBounds(clusterData.bounds)`，随后关闭面板。单地点集合不显示该动作。

默认路径：

```text
集合图钉 -> 集合足迹面板 -> 可选的单条足迹详情
```

辅助路径：

```text
集合图钉 -> 集合足迹面板 -> 在地图中展开
```

## 集合足迹面板

现有 `SamePlaceSheet` 更名并收敛为 `ClusterFootprintSheet`，因为它将承载多个不同地点，不再只是同坐标足迹。

面板内容：

- 标题：`3 个地点`；
- 副标题：`8 条足迹`；
- 多地点时提供带缩放图标的“在地图中展开”动作；
- 足迹按 `createdAt` 倒序，时间相同按 `_id` 稳定排序；
- 每行展示心情、用户、地点、来源、日期和未读状态；
- 点击一行关闭集合面板，并打开对应地图预览/足迹详情；
- 点击遮罩、关闭按钮或按 `Escape` 关闭面板并返回地图。

列表项不再描述为“同一地点”。对话框标签使用“集合足迹”，关闭按钮使用“关闭集合足迹”。面板打开后聚焦关闭按钮，关闭时恢复原焦点。

## 数据刷新与边界处理

- 坐标缺失或非法的足迹不会创建 marker，也不进入集合统计。
- 多条足迹位于同一去重坐标时显示 `1 个地点`，但面板副标题保留真实足迹数。
- `placeCount <= 1` 或 bounds 无效时隐藏“在地图中展开”。
- 足迹刷新后，面板只保留当前可见 ID；全部消失时自动关闭集合选择。
- 来源缺失时回退到全球来源色，心情缺失时使用默认图钉符号。
- `fitBounds` 执行前重新校验 bounds；失败时关闭面板并保持当前地图视野，不阻塞其他地图操作。
- 集合地点数很大时只增加标签文本，不增加叠放图钉数量或改变点击区域。
- 未读圆点只表达“至少一条未读”，不再叠加未读数量。

## 测试策略

### 纯函数

- 坐标按六位小数去重，分别得到地点数和足迹数。
- 来源按频率和固定优先级排序，最多返回三个。
- 未读状态、标签和可访问标签正确。
- 集合 HTML 只叠放最多三枚图钉，并转义动态文本。
- 缓存键覆盖地点数、足迹数、来源序列和未读状态。

### 交互

- 任意集合第一次点击都调用 `openCluster`，不调用 `fitBounds`。
- 集合选择包含正确 ID、bounds、地点数和足迹数。
- 多地点面板点击“在地图中展开”只调用一次 `fitBounds` 并关闭。
- 单地点面板不显示展开动作。
- 列表排序、关闭、Escape、焦点恢复和足迹选择行为正确。
- 数据刷新后移除不可见 ID，列表为空时关闭。

### 回归与视觉

- 单个图钉点击继续打开地图预览。
- 单图钉来源色、未读、选中和脉冲动画不变。
- UI Store 的全局覆盖层清理继续清空集合选择。
- 在 360×800、390×844 和桌面地图验证叠放图钉不遮挡关键控件、标签不溢出、点击区域至少 44px。
- 验证单一来源、混合来源、单地点多足迹、超过三种来源、未读和大地点数状态。

## 验收标准

- 地图集合不再显示裸数字圆球，而显示叠放图钉和 `N 个地点` 标签。
- 集合地点数按坐标去重，面板同时显示真实足迹总数。
- 第一次点击任意集合直接打开集合足迹面板，不发生自动缩放。
- 用户最多再点击一次即可进入单条足迹详情。
- “在地图中展开”是可选动作，且只在多个有效地点时出现。
- 现有地图查询、单图钉和足迹可见性行为没有回归。
